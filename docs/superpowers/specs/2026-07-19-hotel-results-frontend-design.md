# 酒店结果页前端与真机回归设计

## 目标

- 让酒店搜索与实时房型页延续现有 calm utility 前端风格，并具备与餐饮、出行页面一致的 Hero Banner。
- 默认只展示用户决策所需信息，卡片展开后再展示完整详情。
- 删除页面上的调试式标识、内部 ID、原始链接和冗长能力说明，同时保留必要的价格真实性提示。
- 将真实酒店搜索与房型查询加入核心真机回归，完成后合入现有 PR，并把仓库收敛为一个与远程一致的主 worktree。

## 非目标

- 不新增酒店下单、订单查询或模拟预订能力；RollingGo 当前只提供真实搜索与房型报价。
- 不新增图片依赖、组件库或通用卡片抽象。
- 不修改其他场景的视觉设计和业务能力。

## 页面设计

### Hero Banner

- 搜索结果优先使用第一家酒店的真实图片作为背景，并覆盖深色渐变保证文字对比度；图片不可用时使用 CSS 渐变兜底。
- 标题使用当前搜索标题，副标题承载目的地、入住日期和入住人数等真实上下文。
- 角标改为简洁的“酒店 · 实时房价”，不显示 `LIVE HOTEL DATA` 等调试文案。
- Hero 只保留一条简短事实提示：“房价实时更新，以供应商确认页为准。”

### 酒店卡片

- 默认折叠：图片、酒店名、位置/星级、少量核心标签、参考价、展开按钮。
- 展开后：地址、入住条件、设施、取消/价格说明，以及现有“查看实时房型”操作。
- 同一时间只展开一张卡；按钮同步 `aria-expanded`，支持点击卡片标题区展开，并保留明显的按压反馈。
- 隐藏酒店 ID、目的地 ID、Rate ID、原始预订 URL、供应商技术报文和冗长能力边界；这些值仍可保留在动作参数或数据层，避免破坏真实房型查询。

### 房型卡片

- 默认折叠：房型名、价格、床型/餐食摘要。
- 展开后：入住条件、库存、取消政策及必要的价格提示。
- 与酒店卡片复用当前渲染器内的同一套 CSS/JS 展开逻辑，不引入新抽象。

### 动效与无障碍

- 动效只使用 `transform`、`opacity` 和短时长过渡。
- 支持 `prefers-reduced-motion`、`prefers-reduced-transparency` 与 `prefers-contrast`。
- 所有操作保持原生 `button`、可见焦点和至少 44px 的触控高度。

## 数据与错误边界

- `hotel.search` 和 `hotel.detail` 继续使用真实 RollingGo 返回，不制作本地酒店、价格或库存。
- 图片失败时仅隐藏图片并回落到稳定布局，不伪造替代酒店图。
- 真实 provider 错误继续原样进入错误路径，不用空卡片或占位结果掩盖。

## 回归覆盖

- 在核心矩阵增加 C20：
  - Query：`帮我搜索2026年8月8日至10日深圳南山区科技园附近的酒店，2位成人1间房`
  - 预期：`hotel.search` 返回真实酒店列表。
  - 衍生交互：展开酒店卡 → 点击“查看实时房型” → `hotel.detail` → 展开房型卡。
  - 页面证据：Hero、酒店名/地址/参考价、展开状态、实时房型、床型/餐食/取消政策。
  - 安全边界：只读查询，不发起预订。
- 同步 `docs/current-capabilities.md` 的酒店能力和 Query。
- 为现有 smoke runner 最小补齐 `--list-cases` 与 `--core-regression`，核心清单使用矩阵 C01–C20；酒店衍生交互由 runner 自动点击并采集截图、布局和 hilog。

## 验证与交付

- 运行受影响的 Hypium 测试并读取报告；随后运行完整测试，`Failure` 必须为 0，否则停止合入并处理真实失败。
- 构建签名 HAP，安装同一个产物到真机；运行核心回归并确认 `hdc fport ls` 为空。
- 至少保留酒店折叠、展开、实时房型和房型展开截图，以及对应的布局、工具调用和 hilog 证据；核心范围不得存在 `BLOCKED` 或 `NOT_RUN`。
- 合入前 fetch 远程并处理主分支最新改动；推送并合入 PR #64。
- 最终切回并快进本地 `main` 到 `origin/main`，删除过时 worktree、本地场景分支和已合入的远程功能分支，只保留主 worktree 与 `main`。
