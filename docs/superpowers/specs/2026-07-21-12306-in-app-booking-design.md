# 12306 最低价与 App 内购票设计

日期：2026-07-21
状态：已确认

## 目标

在现有 `train.search` 高铁场景中展示 12306 返回的真实最低可售价格，并从展开卡片进入 App 内的 12306 官方购票页面。界面沿用当前高铁卡片和结算 WebView，不新增支付 provider、乘车人表单或第二套高铁页面。

## 已确认的产品行为

- 折叠态只增加 `¥xxx起`，不显示席别或额外说明。
- 最低价仅在当前有票席别中计算；无真实价格时不展示价格。
- 展开态主按钮为“去购票”。
- 点击后在 App 内打开 12306 官网，并预填日期、出发站和目的地。
- 12306 当前没有公开、稳定的指定车次购票深链；用户需要在官网重新选择车次。
- 官网强制唤起 12306 App、微信或支付宝时允许经过验证的外部跳转。
- 关闭购票页后返回原卡片，只提示“订单与支付结果请以 12306 为准”，不推断订票或支付成功。

## 官方能力边界

12306 的公开查询 URL 支持日期、出发站和目的地等查询条件。当前官网中每趟车的“预订”按钮是页面内 JavaScript 动作，依赖实时 `secretStr`、登录会话和 POST 请求，不能转换为稳定的指定车次链接。

铁路票款不经过本项目现有的 Stripe、PayPal 或其他支付能力。登录、乘车人、订单和付款均由 12306 官方页面处理，App 不读取或保存这些数据。

- [12306 官方查询页面脚本](https://kyfw.12306.cn/otn/resources/merged/queryLeftTicket_end_js.js?scriptVersion=1.95041)

## 架构

复用现有链路：

```text
train.search
  -> ToolGatewayClient 解析 12306 余票与最低价
  -> A2uiTrainData
  -> HtmlHomeSnapshot
  -> HtmlTravelHomeRenderer 高铁卡片
  -> train.booking.open
  -> CheckoutWebOverlay
  -> 12306 官方查询、登录、下单与支付页面
```

不新增工具 ID、支付后端、页面路由或 provider adapter。`train.search` 仍是只读查询工具；`train.booking.open` 是客户端导航动作，不表示订单创建。

## 数据模型

`A2uiTrainData` 增加两个可选字段：

```text
price?: string
bookingUrl?: string
```

- `price` 是已格式化的人民币最低可售价格，不带 `¥` 和“起”。
- `bookingUrl` 是 12306 官方查询页 URL，包含本次查询的日期、出发站名称与代码、目的地名称与代码。
- 两个字段均为可选，保持旧响应、旧快照和失败结果兼容。

## 最低价解析

现有余票响应中索引为 39 的字段 `yp_info_new`（按 0 起算）已被 12306 官网用于显示价格，但当前项目未解析。按官网当前逻辑：

1. 将字段按每 10 个字符拆分。
2. 每段首字符是席别代码，第 2 至第 6 个字符是以角为单位的价格整数。
3. 使用余票记录中的对应席别状态判断是否当前可售。
4. `有` 或正整数视为可售；空值、`--`、`无`、`候补`、`*` 和无法识别的值不参与计算。
5. 对可售席别价格取最小值，用整数角格式化，去除无意义的 `.0`。

价格解析失败只使 `price` 为空，不影响车次、时刻和余票展示。不得从车次、里程、历史票价或其他结果估算价格。

## 官方购票 URL

在 `train.search` 已解析出日期和站点代码后生成官网查询 URL，使用 12306 当前公开页面采用的 `date`、`fs`、`ts`、`flag` 和 `linktypeid` 参数。所有车次卡片可共享同一线路查询 URL，因为官网没有公开的车次级深链。

初始 URL 必须同时满足：

- 协议为 `https`；
- 主机严格等于 `kyfw.12306.cn`；
- 路径严格等于 `/otn/leftTicket/init`。

任一条件不满足都不打开 WebView。

## 卡片交互

### 折叠态

保留现有来源标签、车次、时刻、站点、耗时、颜色、间距、圆角和展开按钮。仅在现有耗时信息旁增加同风格的 `¥xxx起`。`price` 为空时不创建价格节点，不显示占位文案。

### 展开态

保留现有详情和“固定到桌面”能力，增加主动作：

```text
id: train.booking.open
label: 去购票
kind: client
args.bookingUrl: 经校验的 12306 URL
```

“去购票”排在首位并使用当前高铁主色；“固定到桌面”继续作为次要动作。点击按钮不得触发卡片收起。

当前 Home 高铁场景始终使用 `HtmlTravelHomeRenderer`，因此本次不修改未使用的原生 `TrainOptionsView`。

## App 内购票层

复用 `CheckoutWebOverlay` 和 `A2uiWebSurfaceView`，为 `CheckoutWebKind` 增加 `train`：

- 标题为“12306 官方购票”；
- 顶部显示当前页面实际域名；
- 保留返回、加载、主页面错误和新窗口能力；
- JavaScript 与 DOM Storage 保持开启，以满足官网登录和订单流程；
- 关闭后恢复原高铁卡片并显示结果边界提示。

官网内部 HTTPS 页面留在 WebView。自定义 Scheme 默认阻止；只有真机流程中实际观察并验证过的支付或身份核验 Scheme 才加入固定白名单，不允许通用自定义 Scheme 透传。

## 动作与安全策略

- `train.booking.open` 必须存在于当前渲染文档，继续受现有 HTML 动作策略约束。
- `Index` 在打开购票层前再次验证 `bookingUrl`，不信任 WebView 传回的字符串。
- 远程页面不得触发项目工具调用、支付状态写入或订单状态更新。
- App 不注入账号、密码、乘车人或支付信息，不解析官网 DOM 获取订单结果。
- 非 HTTPS 跳转、伪造 12306 域名和未验证 Scheme 均阻止并记录不含敏感数据的日志。

## 状态与失败处理

| 情况 | 行为 |
| --- | --- |
| 价格字段有效且存在可售席别 | 显示 `¥xxx起` |
| 价格缺失、格式变化或无可售席别 | 不显示价格，余票卡片继续可用 |
| 购票 URL 缺失或校验失败 | 不打开购票层，保留当前卡片 |
| 官网加载失败 | 使用现有 WebView 错误页，可返回卡片 |
| 官网要求外部 App 核验或支付 | 仅放行已验证 Scheme，并明确这是官网流程 |
| 用户关闭购票层 | 返回原卡片，提示以 12306 订单和支付结果为准 |
| 官网页面看似显示成功 | App 不据此生成本地成功状态 |

## 验证

### 自动测试

1. 12306 价格解析：覆盖最低可售价格、较低但无票席别、候补、异常长度、非数字价格和空字段。
2. A2UI/HTML 快照：验证 train 数据携带价格与官方 URL；旧数据仍可解析。
3. Renderer：验证折叠态只有 `¥xxx起`，不含席别；展开态包含“去购票”；航班和其他卡片不受影响。
4. 动作策略：验证文档内合法动作可通过，伪造动作、伪造域名、HTTP 和错误路径被拒绝。
5. WebView 导航决策：覆盖 12306 页面、HTTPS 页面、未验证 Scheme 和关闭返回，不产生订票成功状态。
6. 使用 `scripts/run-hvigor-tests-strict.mjs`，以 Hypium `test_result.txt` 为准，不以 shell 退出码代替断言结果。

### 真机验收

1. 安装新构建到 HarmonyOS 真机，确认 `hdc fport ls` 为空，证明不依赖 Mac gateway。
2. 发起真实高铁查询，核对卡片价格来自同次 12306 响应。
3. 验证折叠态、展开态和“去购票”按钮样式与现有页面一致。
4. 验证 App 内打开带日期和站点的 12306 查询页，可以继续进入登录、车次选择和支付页面。
5. 验证返回后没有虚假订单或支付成功状态。

自动验收停在不可逆下单之前。实际登录、身份核验和付款只能由用户本人执行；若官网强制外部 App，则按已确认边界如实外跳，不能声称全程留在 WebView。

## 完成标准

- 真实有价车次在折叠态显示正确的 `¥xxx起`，无真实价格时不显示。
- 展开态“去购票”只打开经过校验的 12306 官方查询页。
- 原有高铁查询、筛选、展开、固定桌面和航班展示无回归。
- App 不创建铁路订单、不处理铁路票款、不伪造订单状态。
- 严格 Hypium 测试通过，真机查询与 WebView 跳转证据完整。

## 不在本次范围

- 直接提交指定车次订单；
- App 内乘车人、席别、优惠票或联系人管理；
- 自建铁路支付或接入 Stripe、PayPal；
- 抢票、候补提交、退票、改签和订单同步；
- 根据 WebView DOM 推断或抓取订单结果；
- 重做高铁卡片或新增独立购票页面。
