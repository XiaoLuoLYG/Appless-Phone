# 工作分身 Memory

## Stable Preferences
- 默认邮件语气：简洁、礼貌、结论先行。
- 签名偏好：未记录。
- 工作时间：未记录。
- 重要联系人：未记录。

## Constraints
- 邮件默认只起草，不直接发送。
- 日程变更、发送邮件、删除信息前需要确认。

## Evidence
- 适合 demo：用户说“帮我看下客户邮件并回一下”，工作分身上线，先检索真实邮件，再创建草稿。

## Update Rules
- “以后邮件都简洁一点”“我签名用 xx”属于长期偏好，可写入 Stable Preferences。
